# Roadmap

## À faire avant version 1.0

- Stabiliser les règles d’initiation quadratique après retours de parties bêta.
- Vérifier régulièrement la cohérence entre règles Markdown, deck Markdown, `rules.js` et `cards.js`.
- Ajouter une vraie pile de réaction pour les Interrupts si les tests de partie montrent un besoin clair.
- Finaliser une fiche de mise en place rapide pour élèves.
- Compléter les tests moteur sur les cas limites les plus fréquents.

## Idées futures

- Ajouter un historique exportable pour analyse pédagogique.
- Ajouter des variantes de difficulté pour classes ou ateliers courts.
- Préparer des supports d’animation pour enseignants.

## Mode multijoueur distant avancé

- Clarifier le périmètre : prototype simple, salons avec code court, ou vrai mode temps réel.
- Ajouter des salons de partie avec code court si l’usage en classe le justifie.
- Étudier une authentification légère ou des restrictions d’accès pour les hébergements publics.
- Garder l’API PHP/MySQL actuelle comme prototype pédagogique tant que le besoin avancé n’est pas validé.

## Mains pédagogiques

- Préparer des mains de départ prêtes à poser.
- Documenter l’objectif pédagogique de chaque main.
- Ajouter des scénarios guidés : overflow, reboot, parasites, choix de profondeur, remontée.

## PNG par carte

- Ajouter un PNG par carte dans `physique/initiation/impression/png/`.
- Conserver les PDF recto/verso comme kit d’impression principal.
- Vérifier que les exports restent cohérents avec le deck Markdown.

## Mode tutoriel

- Ajouter un mode tutoriel dans la V2.
- Introduire progressivement les phases, la pile d’appels, le cas de base et l’overflow.
- Réutiliser les mains pédagogiques plutôt que promettre une IA ou un parcours complet dès maintenant.
